# Procedures Index

Use this directory for operational procedures that agents and humans must follow while running project management inside `.pm/`.

## Core procedures

- `.pm/procedures/pm-operations.md` - assisted day-to-day PM operating loop, authority matrix, escalation triggers.
- `.pm/facets/pm.governance.core/procedures/PM_WEEKLY_KPI_REVIEW_CHECKLIST_V1.md` - weekly KPI rubric/checklist for PM-loop quality review and corrective action tracking.

## Related SOPs outside `.pm/`

- `docs/sop/PM_SCHEMA_MIGRATION_SOP.md` - baseline `.pm` schema migration procedure.
- `docs/sop/PM_TASK_SCHEMA_V2_MIGRATION_SOP.md` - migration from backlog-table canonical tasks to file-native canonical tasks.

## Usage expectation

Before making PM metadata changes, agents should:
1. Read `.pm/procedures/pm-operations.md`.
2. Read any operation-specific procedure(s) relevant to the requested action.
3. Follow approval/escalation rules when procedures or policies indicate approval is required.
